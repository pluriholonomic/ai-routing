# SM2 attempt-limit amendment

**Written:** 2026-07-18 after the first synthetic screening run
**Scope:** unreliability treatments only
**Reason:** repair a missing economic margin, not select a favorable sign

The first SM2 screening allowed every request to attempt both providers. With
provider-request technical shocks held fixed across router rules, final success
then depended only on whether at least one provider succeeded, not on their
order. As a result, changing failure loss could not change a paired router
contrast. The low-loss and high-loss results were exactly duplicated.

This was a valid null diagnostic but did not test the registered reliability
question. Before rerunning, the low- and high-loss unreliability scenarios are
amended to permit one route attempt. This represents a deadline-constrained or
no-fallback request and makes first-provider reliability allocative. All other
provider costs, quotes, capacity, latency, reliability, seeds, router rules,
outcomes, and claim boundaries remain unchanged.

The original generated files are overwritten only in the working analysis
directory; this amendment records the reason and the first-run qualitative
result. The revised output remains synthetic screening evidence and cannot be
called confirmatory.
