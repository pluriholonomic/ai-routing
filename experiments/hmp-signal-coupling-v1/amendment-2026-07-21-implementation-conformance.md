# WF-18 implementation-conformance amendment

Date: 2026-07-21 UTC.

Before remote deployment, a local development run used Hugging Face revision
`aeb7f165669888514548170606fd3a29b1d0fb5f`. It had 6 mechanism days, 19
provider-pair/model clusters, and 48 delegated choices. The primary excess
residual-coupling estimate was `0.0187425` above the circular-shift null with a
one-sided randomization `p = 0.10995`. One pair supplied 76.7% of paired events.
SC2 and SC3 were power-gated. No confirmatory release or immutable release marker
was created.

That development run exposed implementation deviations from the already-written
method section, not favorable result-dependent specifications. Before the first
remote run, this amendment therefore:

1. replaces the same-sample routing-SNR fit with leave-date-out predictions;
2. requires a complete Holm-adjusted SC1--SC4 family before any claim promotion;
3. tests whether quote coupling predicts the magnitude of the elasticity wedge,
   instead of treating any estimable wedge as success;
4. adds the registered secondary windows, direction, frequency, different-model,
   provider-identity, enforcement-window, and leave-one-unit diagnostics; and
5. changes release language from false “first outcome access” language to an
   accurate one-time support-gated promotion marker.

The ordered hypotheses, primary 24-hour statistic, 2,000 primary permutations,
support thresholds, 20% concentration cap, alpha, and claim boundaries did not
change. The development estimate above remains disclosed in every execution
report. Public quote outcomes remain visible during accrual; the protection
against optional promotion is executable fixed support and multiplicity gates,
not blinding.
