# Strategic-routing publication completion audit

**Date:** 2026-07-18  
**Objective audited:** execute and improve the strategic-simulation plan until

rigorous economic result.

## Bottom line

The publication objective is complete for a deliberately narrow ACM EC paper.
The current result is not the full Version 1 market-simulation platform
described in the original eight-week execution plan.

The accepted object is:

> A router-imposed finite cut penalty creates an exact delayed-credit region.
> In a calibrated binary-price, fixed-rival MDP, a path-equivalent commitment
> option preserves every primitive price path and the exact provider value,
> yet it changes the path found by a frozen tabular-Q learner. The intervention
> helps inside the rational cut region and harms beyond it.

Round 4 recommends weak accept with confidence 5/5 under an ACM EC lens.

## Original round-1 acceptance package

The project changed contributions after the early high-price interpretation
failed its deviation audit. The original package and the final disposition are:

1. **Optimal or welfare-bounded stateful steering theorem:** not delivered.
   The paper no longer makes a welfare-optimal-router claim. The replacement is
   an exact rational-memory boundary and path-equivalent provider-value theorem.
2. **Confirmatory E-SIM3 and E-SIM4:** executed, then narrowed because the
   high-price path was not an equilibrium. E-SIM5--9 test and falsify the
   surviving delayed-credit mechanism.
3. **Separate aggregate-demand and routed-share elasticity:** satisfied in the
   manuscript. Routed share is generated by the router; aggregate demand is an
   external sensitivity and is not estimated from routed share.
4. **Provider type and capacity regime:** partial. E-SIM1 uses provider species
   and cost bands, but the final paper makes no reserved-capacity conclusion.
5. **Executable open-source-router conformance:** satisfied for LiteLLM
   selection semantics by SM4. Service-path and llm-d fidelity remain open.
6. **Claim-type separation:** satisfied by the paper, result registry,
   evidence tests, and this ledger.

The old package was a path to a different welfare-and-steering paper. A failed
equilibrium audit correctly caused a scientific pivot rather than threshold
shopping.

## Remaining research program

The next paper, rather than a condition for this paper, should execute the
unresolved general-equilibrium branch:

1. implement the PettingZoo parallel environment and independent PPO/SAC;
2. endogenize at least two provider price/capacity policies;
3. preregister best-response exploitability and welfare accounting;
4. fit low/base/high capacity bands by provider type;
5. run the full router-mechanism factorial and router-as-leader optimization;
6. validate request-serving behavior through llm-d Inference Sim;
7. transport the primary welfare comparison across model, workload, load, and
   provider-type holdouts.

None of those steps should be retrofitted into the present paper's claim.
