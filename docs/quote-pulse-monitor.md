# Public quote-pulse monitor

## Purpose and boundary

H67 detects a narrow, public quote-dynamics signature at the retained
five-minute endpoint cadence:

\[
\text{quote cut} \rightarrow \text{simulated top-tier entry} \rightarrow \text{quote reversion}.
\]

It is a screening tool for **quote-pulse candidates**. The share movement is
mechanically generated by the documented inverse-square public-price rule; it
does not establish actual selected-provider flow, strategic intent, profit,
MEV, front-running, or customer harm.

An allocation-capture claim needs the separate, owned, payload-free route
telemetry contract: quote/candidate version, selected provider, retry chain,
terminal outcome, latency, and billed-token fields. It must not contain prompts,
completions, account identifiers, or credentials.

## Definitions fixed before analysis

The pre-committed thresholds live in `config/quote_pulse.toml`.

| Object | Definition |
|---|---|
| primary scenario | `short_chat`; other fixed request shapes are robustness views, not independent events |
| cut | public expected request quote declines by at least 5% between contiguous snapshots |
| contiguous | adjacent snapshots are no more than 10 minutes apart |
| top tier | one or more providers share the maximum simulated share |
| unique leader | exactly one provider occupies the top tier |
| fade | quote returns within 2% of its pre-cut level within a fixed horizon |
| horizons | 30, 60, 120, 240, 360, and 1,440 minutes |

A provider withdrawal, candidate-set change, missing observation, or unobserved
follow-up is not called a fade. Those cases remain separately visible in the
quality ledger.

## Data construction

The hourly GitHub Actions capture already takes eleven five-minute endpoint
snapshots. H67 replays `orcap.routing_simulation.simulate_snapshot` over each
retained endpoint timestamp for the fixed 20-model panel. This preserves the
five-minute quote clock without sending prompts or buying inference.

If endpoint snapshots are unavailable, H67 falls back to the saved 15-minute
`routing_simulation` surface and labels the source explicitly. The fallback is
not suitable for ruling out sub-15-minute fades.

## Outputs

| Artifact | Grain | Meaning |
|---|---|---|
| `h67_quote_pulse_surface` | timestamp × model × scenario × provider | replayed public price/simulated-share surface with tie-aware states |
| `h67_quote_pulse_paths` | contiguous provider transition | scenario-level cuts, top-tier state, and right-censored horizon outcomes |
| `h67_quote_pulse_events` | primary-scenario provider-model episode | de-duplicated quote-cut episode; scenarios never inflate event count |
| `h67_quote_pulse_quality` | event | candidate-set and follow-up inclusion ledger |
| `h67_quote_pulse_scorecard` | provider | descriptive cut, candidate, and fade counts |
| `h67_quote_pulse_dashboard.html` | latest analysis run | locally filterable event ledger |
| `h67_summary.json` | analysis run | evidence status, coverage gates, and claim boundary |

Run locally against hydrated endpoint data:

```bash
ORCAP_ANALYSIS_SOURCE=local uv run orcap analyze --hypothesis h67 --out analysis
```

The hourly route-simulation workflow runs H67 after H43 and H66 over its
latest 26-hour artifact mirror. That run is useful for candidate detection and
short-horizon follow-up only. The existing nightly full reanalysis hydrates the
HF history and supplies the seven-day/30-day panel. GitHub can insert gaps
between capture jobs, so an event must pass the per-horizon contiguous-followup
check; a missing future is right-censored, never counted as no fade.

The source definition is versioned locally; publishing or changing remote
schedules requires a separate operator action.

## Interpretation and stop rules

The public screen is `insufficient_temporal_coverage` until it has seven days
and 80 independent primary-scenario cut episodes. It is
`no_public_quote_pulse_and_fade_observed` only after that gate passes and no
eligible candidate fades in the pre-registered longest horizon. It remains a
public quote-dynamics result even when candidates are observed.

Do not report an MEV-like allocation claim unless owned route-attempt telemetry
shows a selected-provider share increase after a candidate event, with
pre-trends and matched controls. Do not report harm without a separate
successful-completion, retry, latency, or cost result.
