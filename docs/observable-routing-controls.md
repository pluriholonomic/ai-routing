# Observable routing controls and decentralized comparators

## Role in the research program

Transparent or partially transparent decentralized venues are an external
laboratory for constructing and falsifying routing-enforcement signatures.  A
result there is not evidence that a centralized LLM router behaves the same
way.  Each venue must be classified by the unit it actually reveals.

| Venue / current collector | Observable unit | Useful control | Not identified |
|---|---|---|---|
| Livepeer Gateway (H51) | public five-minute regional swap/reuse message counts | broad routing-adjustment pressure and temporal clustering | selected orchestrator, stream, price, delivery, or provider allocation |
| Akash open market (H55/H61/H62) | block-pinned current open GPU bids and aggregate capacity fields | supply/quote availability and repeated capacity shocks | lease-level delivery, demand, or request routing |
| Chutes (H53) | public cumulative invocation counters by chute plus configured deployment fields | aggregate workload growth versus configured supply | selected worker, successful completion, or user flow |
| Nosana (H58/H59) | declared node registry and source-defined market job-state counts | visible participation and aggregate job-state dynamics | job-to-node assignment or execution quality |
| CoW / Uniswap (H41/H49/H52/H56/H57) | finalized on-chain execution, quote, and depth primitives | validate event-study and latency-arbitrage methodology where execution is observable | inference-provider behavior or LLM routing quality |

## Use in an indirect information-advantage screen

For every centralized-router quote event, retain a matched time block in each
available external control.  A candidate is less consistent with a
router-specific information advantage if it coincides with a broad decentralized
compute capacity/demand disturbance.  A decentralized execution result can
validate that the event-study code detects a pre-registered public transition;
it cannot transfer a front-running finding across market designs.

## Current coverage gate

The current H41 comparator panel is still coverage-gated.  It has one observed
day for the existing Akash/Chutes/DeFi sources and does not yet have a
multi-day provider-selection or job-assignment tape.  The next valid expansion
is repeated finalized, source-specific observations—not treating bid,
invocation, registration, or aggregate-state data as realized allocation.
